# CHANGE MANAGEMENT

## 1. Purpose
Ensure all modifications to the system are intentional, reviewed, and reversible.

## 2. Change Types
- Code changes
- Configuration updates
- Dependency updates
- Architecture modifications
- Safety invariant changes

## 3. Requirements
- Every change must have a reason
- Every change must be reviewed
- Every change must be documented
- Every change must be test‑validated
- No emergency changes without post‑review

## 4. Approval Flow
1. Author prepares change
2. Self‑review
3. Safety review
4. Architecture review
5. Merge approval
6. Deployment

## 5. Rollback Rules
- Rollback must always be possible
- Rollback must be tested
- Rollback must be documented

## 6. Philosophy
Change is power — and power must be controlled.
